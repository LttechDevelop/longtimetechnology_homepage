## 一、系統架構概覽

官網是純靜態網站（HTML / CSS / JavaScript），沒有後端伺服器，也不需要資料庫。所有檔案都放在 GitHub repository，並透過 **GitHub Actions** 自動部署到 **GitHub Pages**。

| 項目 | 說明 |
|------|------|
| Repository | `https://github.com/boatman3132/longtimetechnology` |
| 正式網域 | `www.lttech.com.tw` |
| 主要分支 | `main` |
| 部署方式 | 推送到 `main` → GitHub Actions 自動建置並上線 |

**你需要有這個 GitHub repository 的寫入權限才能維護官網。** 請向負責人申請 collaborator 或 organization 成員資格。

## 二、本機開發環境準備

只有在更新「最新消息」時才需要在本機操作，其餘 CSV 修改直接在 GitHub 網頁上也能完成。

### 環境需求

- **Node.js 20+**（可從 [nodejs.org](https://nodejs.org) 下載）
- **Git**

### 初次設定

```bash
# 1. 將 repository clone 到本機
git clone https://github.com/boatman3132/longtimetechnology.git
cd longtimetechnology

# 2. 安裝相依套件（只需做一次）
npm install
```

## 三、更新最新消息

最新消息的運作方式：每則新聞是 `/news/` 資料夾下的一個 Markdown（`.md`）檔案，執行 build 指令後會自動產生 `/dist/news-index.json`，網頁讀取這個 JSON 來顯示新聞列表。

### 3.1 新增一篇新聞

#### 步驟 1：建立 Markdown 檔案

在 `/news/` 資料夾新增一個檔案，**命名規則嚴格要求**：

```
YYYY-MM-DD文章標題.md
```

範例：
```
2026-07-10公司六月營收.md
2026-07-15與某公司簽署MOU.md
```

> 可以直接複製 `/news/最新消息空白樣本` 這個檔案作為起點，改名後填入內容即可。

#### 步驟 2：填寫檔案開頭（Frontmatter）

每個 `.md` 檔案最上方是用 `---` 包起來的 YAML 區塊，需要填寫四個語言版本的標題、標籤、圖片：

```yaml
---
titles:
  tw: "LTT 6 月營收 XXXX 萬元，年增 XX%"
  cn: "LTT 6 月营收 XXXX 万元，同比增长 XX%"
  en: "LTT June Revenue Reaches NT$XX Million, Up XX% YoY"
  jp: "LTT 6月売上高はXXXX万台湾ドル、前年同月比XX%増"

tags:
  tw: ["營收數據"]
  cn: ["营收数据"]
  en: ["Revenue Data"]
  jp: ["売上データ"]

images:
  - "news/photo/2025-10-07公司營運成長示意圖.jpg"
---
```

**常用標籤對照：**

| 類型 | tw | cn | en | jp |
|------|----|----|----|----|
| 每月營收 | 營收數據 | 营收数据 | Revenue Data | 売上データ |
| 公司公告 | 公司消息 | 公司消息 | Company News | 会社ニュース |
| 產業新聞 | 產業動態 | 产业动态 | Industry News | 業界動向 |
| 員工活動 | 員工活動 | 员工活动 | EmployeeEvents | 社員活動 |

#### 步驟 3：放入新聞圖片（如有）

將圖片放到 `/news/photo/` 資料夾，建議命名加上日期前綴，例如：

```
2026-07-10六月營收說明圖.jpg
```

圖片放好後，在 frontmatter 的 `images` 欄位填入路徑：

```yaml
images:
  - "news/photo/2026-07-10六月營收說明圖.jpg"
```

若無圖片，可使用現有通用圖片（營收類新聞常用）：

```yaml
images:
  - "news/photo/2025-10-07公司營運成長示意圖.jpg"
```

#### 步驟 4：撰寫四語言內文

Frontmatter 結束後，用語言標記區隔各語言版本，格式如下：

```markdown
<!-- lang:tw -->

（繁體中文內容）

---

<!-- lang:cn -->

（简体中文内容）

---

<!-- lang:en -->

（English content）

---

<!-- lang:jp -->

（日本語コンテンツ）

---
```

每個語言區塊之間以 `---` 分隔。各語言內容可以不完全相同，但建議資訊一致。

#### 步驟 5：執行 Build 指令

新聞 Markdown 寫好後，需要執行一次 build 指令來更新 JSON 索引：

```bash
npm run build:news
```

成功後終端機會顯示類似：

```
[build-news-index] Generated 35 entries at dist/news-index.json
```

這個指令會讀取 `/news/` 下所有 `.md` 檔，自動產生 `/dist/news-index.json`，包含標題、日期、摘要等，不需要手動編輯 JSON。

#### 步驟 6：提交並推送到 GitHub

```bash
git add .
git commit -m "新增 2026-07-10 六月營收新聞"
git push origin main
```

推送後，GitHub Actions 會自動執行（約 1–2 分鐘），完成後官網即更新。可以到 GitHub repository 的 **Actions** 頁籤確認部署狀態是否為綠色勾勾。

---

### 3.2 修改或刪除現有新聞

**修改**：直接編輯對應的 `.md` 檔，重新執行 `npm run build:news`，推送即可。

**刪除**：刪除對應的 `.md` 檔（以及不再使用的圖片），執行 `npm run build:news`，推送即可。

## 四、更新營收與股價圖表

投資人關係頁面（`/investors/`）有一張「營收與股價趨勢圖」，資料來源是這個 CSV 檔：

```
/investors/榮炭營收股價成交量.csv
```

### 4.1 CSV 格式說明

```csv
年度/月份,當月營收,股價,股票成交量
2026/6,XXXXX,XX.XX,"X,XXX,XXX"
2026/5,29920,20.65,"4,668,704"
2026/4,35798,25,"15,381,486"
...
```

| 欄位 | 說明 | 單位 |
|------|------|------|
| 年度/月份 | 格式為 `YYYY/M`，月份不補零 | — |
| 當月營收 | 當月合併營收 | 新台幣千元 |
| 股價 | 當月最後一個交易日的收盤價 | 新台幣元 |
| 股票成交量 | 當月累計成交股數 | 股 |

### 4.2 更新步驟

每月月初（通常月營收公布後），在 CSV 最上方**新增一行**（最新資料放最上面）：

```csv
年度/月份,當月營收,股價,股票成交量
2026/6,41500,22.30,"3,500,000"    ← 新增這行
2026/5,29920,20.65,"4,668,704"
...
```

**資料來源：**

- **當月營收**：公司每月 10 日前公告於公開資訊觀測站（MOPS）
  - 網址：https://mops.twse.com.tw/mops/#/web/t146sb05?companyId=6555
- **股價**：查詢台灣證券交易所，股票代號 **6555**，取當月最後一個交易日收盤價
  - TWSE 月K資料：https://www.twse.com.tw/rwd/zh/afterTrading/STOCK_DAY?stockNo=6555
- **成交量**：同上，取當月累計成交股數（或從 TWSE 月報表取得）

### 4.3 推送上線

```bash
git add investors/榮炭營收股價成交量.csv
git commit -m "更新 2026 年 6 月營收股價資料"
git push origin main
```

CSV 本身不需要 build，推送後 GitHub Actions 直接部署即可。

## 七、常見問題

**Q：推送後網站沒有更新？**  
到 GitHub repository → Actions 頁籤，確認最新的 workflow run 是否成功（綠色勾勾）。若失敗，點進去看錯誤訊息。

**Q：`npm run build:news` 報錯？**  
最常見原因是新聞檔案命名不符規則（前十個字元必須是 `YYYY-MM-DD` 格式）。確認檔名後重新執行。

**Q：新聞出現但圖片不顯示？**  
確認圖片已放入 `/news/photo/`，且 frontmatter 裡的 `images` 路徑拼寫正確（包含 `news/photo/` 前綴）。

**Q：想修改網站其他頁面的內容？**  
直接編輯對應的 `.html` 檔案，不需要 build，直接 `git push` 即可上線。
